Hotel Pricing Classification Pipeline

Dari raw booking data → feature extraction → clustering → klasifikasi tipe pelanggan → promo assignment

1
Raw data ingestion
PMS / Channel Manager / Booking Engine
Input

Sumber data utama adalah Property Management System (PMS) dan channel manager. Setiap baris = satu transaksi pemesanan. Minimal membutuhkan 5 kolom.

booking_date
DATE — tanggal booking dibuat
arrival_date
DATE — tanggal check-in
departure_date
DATE — tanggal check-out
room_revenue
FLOAT — total revenue kamar (IDR)
room_type
STRING — kategori kamar
booking_channel
STRING — OTA / direct / corporate
Data minimal 12 bulan historis untuk menangkap pola musiman. Data 24+ bulan ideal untuk pelatihan model klasifikasi yang robust.
2
Komputasi tiga penciri utama
LOS · Booking Window · ADR
Primary features

Tiga penciri ini adalah fondasi seluruh pipeline. Dihitung langsung dari kolom raw data — tidak membutuhkan sumber eksternal.

LOS = (departure_date - arrival_date).days BW = (arrival_date - booking_date).days ADR = room_revenue / LOS
LOS — Length of Stay
Durasi menginap (hari). Range: 1–30+
Transient: 1-2 · Short: 3-5 · Leisure: 6-13 · Extended: 14+
BW — Booking Window
Lead time pemesanan (hari). Range: 0–365
Walk-in: 0-1 · Short: 2-7 · Mid: 8-30 · Long: 31-90 · Ultra: 90+
ADR — Average Daily Revenue
Revenue per malam (IDR). Proxy WTP tamu
Budget: <P25 · Standard: P25-P50 · Premium: P50-P75 · Luxury: >P75
3
Feature engineering — fitur turunan & interaksi
6 fitur baru dari kombinasi LOS · BW · ADR
Derived features

Kombinasi antar penciri menghasilkan sinyal perilaku yang tidak tertangkap oleh masing-masing fitur secara terpisah. Urgency Ratio adalah fitur paling diskriminatif antar tipe pelanggan.

Total_Revenue = ADR × LOS Urgency_Ratio = ADR / max(BW, 1) # WTP vs lead time RevPAB = ADR × LOS / max(BW, 1) # Revenue per Advance Booking day LOS_BW_Ratio = LOS / max(BW, 1) # Stay intensity per booking day ADR_per_BW = ADR / max(BW, 1) # Price urgency per day Lead_Urgency = 1 / (1 + exp(-BW / λ)) # λ=14, sigmoid decay urgency score
Total Revenue
Nilai absolut per booking. Menangkap kombinasi harga × durasi.
Urgency Ratio ★
Makin tinggi = makin inelastis terhadap harga. Kunci identifikasi Last-Minute Premium.
RevPAB
Revenue normalized terhadap waktu booking. Fitur terpenting untuk clustering.
Lead Urgency
Score 0–1. Mendekati 1 = last-minute. Mendekati 0 = early planner.
4
Komputasi fitur entropi (Shannon)
H(LOS) · H(BW) · H(ADR|LOS) · I(LOS;BW)
Entropy features

Entropi dihitung per segmen (per property, per bulan, per BW bin) menggunakan rolling window 30 hari. Bukan per-booking, melainkan sebagai karakteristik distribusi konteks di mana booking terjadi.

H(X) = -Σ p(xᵢ) × log₂(p(xᵢ)) # Shannon entropy dasar H_LOS = H(distribusi LOS dalam rolling window) H_BW = H(distribusi BW dalam rolling window) H_ADR_LOS = H(ADR, LOS) - H(LOS) # Conditional entropy MI_LOS_BW = H(LOS) + H(BW) - H(LOS, BW) # Mutual information # Implementasi Python: from scipy.stats import entropy H_BW = entropy(np.histogram(bw_window, bins=5)[0], base=2)
H(BW) — booking pattern entropy
Tinggi → pola tak terprediksi → trigger dynamic pricing. Rendah → konsentrasi di satu window → advance discount cukup.
H(LOS) — stay duration entropy
Tinggi → mix transient + leisure → butuh LOS-based pricing. Rendah → homogen → satu rate structure cukup.
H(ADR|LOS) — conditional entropy
Rendah → LOS prediktor kuat untuk ADR. Tinggi → perlu fitur tambahan untuk prediksi harga.
I(LOS;BW) — mutual information
Tinggi → book jauh cenderung stay lama → bundle pricing dan cross-prediction bernilai tinggi.
Entropi digunakan sebagai fitur kontekstual dalam klasifikasi — bukan sebagai fitur clustering langsung, karena entropi mengkarakterisasi distribusi, bukan satu booking individual.
5
Fitur temporal & diskretisasi
DoW · Season · Holiday · LOS_cat · BW_cat · ADR_tier
Temporal + categorical

Fitur waktu menangkap permintaan periodik. Diskretisasi mengubah nilai kontinu menjadi kategori bermakna bisnis yang dikenali model lebih mudah.

DoW_arrival = arrival_date.dayofweek # 0=Senin, 6=Minggu Month = arrival_date.month # 1–12 Is_Weekend = 1 if DoW in [5,6] else 0 Holiday_prox = min(|arrival - holiday| for holiday in calendar) Season = {1-3:'Q1', 4-6:'Q2', 7-9:'Q3', 10-12:'Q4'}[month] LOS_cat = pd.cut(LOS, bins=[0,2,5,13,999], labels=['transient','short','leisure','extended']) BW_cat = pd.cut(BW, bins=[-1,1,7,30,90,999], labels=['walkin','short','mid','long','ultra']) ADR_tier= pd.qcut(ADR, q=4, labels=['budget','standard','premium','luxury'])
6
Preprocessing — normalisasi & encoding
MinMaxScaler · One-hot encoding · Imputation
Preprocessing

Fitur numerik dinormalisasi sebelum clustering (K-Means sensitif terhadap skala). Fitur kategorik diubah ke numerik untuk algoritma ML.

Numerical → MinMaxScaler: x' = (x - x_min) / (x_max - x_min) Categorical → pd.get_dummies(drop_first=True) # one-hot encoding Missing → median imputation (numerical), mode imputation (categorical) Outlier → IQR clipping: clip to [Q1 - 1.5*IQR, Q3 + 1.5*IQR]
Untuk clustering, hanya fitur numerik yang dinormalisasi (LOS, BW, ADR, Urgency_Ratio, RevPAB, Total_Revenue). Untuk klasifikasi, semua 20+ fitur digunakan termasuk one-hot encoded kategorik.
7
Clustering — K-Means unsupervised
6 fitur numerik · k=6 · silhouette validation
Unsupervised learning

Clustering menggunakan 6 fitur inti yang paling diskriminatif. Output adalah label cluster numerik (0–5) yang kemudian diinterpretasikan secara bisnis untuk diberi nama tipe pelanggan.

Fitur clustering (semua dinormalisasi 0–1): X_cluster = [LOS_norm, BW_norm, ADR_norm, Urgency_Ratio_norm, RevPAB_norm, Total_Revenue_norm] Algoritma: KMeans(n_clusters=6, init='k-means++', n_init=10, random_state=42) Validasi pemilihan k: Elbow method: SSE vs k (k=2..10) Silhouette score: pilih k dengan score tertinggi Business interpretation: apakah tiap cluster dapat diberi nama bermakna? Alt algoritma: DBSCAN untuk deteksi outlier / anomali booking
LOS (normalized)
Pembeda utama transient vs extended stay
BW (normalized)
Pembeda planner vs last-minute booker
ADR (normalized)
Pembeda price tier dan willingness to pay
Urgency Ratio
Kunci identifikasi Last-Minute Premium
RevPAB
Nilai booking per advance booking day
Total Revenue
Nilai absolut kontribusi revenue per booking
Output clustering: setiap booking mendapat label cluster 0–5. Tim revenue manager menganalisis karakteristik centroid tiap cluster untuk memberi nama tipe. Proses ini dilakukan SATU KALI untuk membangun ground truth label training data klasifikasi.
8
Training model klasifikasi — Random Forest
20+ fitur · 6 kelas target · Stratified K-Fold
Supervised learning

Label dari clustering digunakan sebagai target kelas (ground truth). Model klasifikasi dilatih dengan semua 20+ fitur — termasuk entropy, temporal, dan kategorik — untuk dapat melakukan prediksi real-time pada inquiry baru.

Target : customer_type ∈ {0:Strategic Planner, 1:Business Traveler, 2:Last-Minute Premium, 3:Deal Seeker, 4:Extended Stay, 5:Occasion Traveler} Features : [LOS, BW, ADR, Total_Revenue, Urgency_Ratio, RevPAB, LOS_BW_Ratio, Lead_Urgency, H_LOS, H_BW, H_ADR_LOS, MI_LOS_BW, LOS_cat_enc, BW_cat_enc, ADR_tier_enc, DoW_arrival, Month, Is_Weekend, Holiday_prox, Season_enc] Algorithm : RandomForestClassifier(n_estimators=300, max_depth=12, class_weight='balanced', random_state=42) Validation: StratifiedKFold(n_splits=5) Metrics : F1-macro, Precision, Recall per kelas, Confusion Matrix
Mengapa Random Forest?
Tahan terhadap outlier, interpretable via feature importance, tidak sensitif terhadap skala fitur, native multi-class. Alternatif: XGBoost untuk performa lebih tinggi.
Class imbalance handling
Gunakan class_weight='balanced' karena beberapa tipe (Extended Stay, Occasion) volume lebih sedikit. Alternatif: SMOTE oversampling pada minority class.
9
Real-time scoring & promo assignment
Klasifikasi sebelum booking confirmed → promo dipersonalisasi
Output

Model berjalan real-time saat tamu browsing atau saat inquiry masuk. Output adalah tipe pelanggan + promo yang dikirimkan otomatis via channel yang sesuai.

Input : {BW_realtime, LOS_estimated, ADR_proxy, DoW, Month, ...} Output : {customer_type, confidence, promo_type, channel, timing} Expected_Revenue = P(convert | type, promo) × ADR_adj × LOS Promo_ROI = (Revenue_uplift - Promo_cost) / Promo_cost
Feedback loop: setiap booking yang selesai dikembalikan sebagai data training baru. Model diretrain mingguan. Lakukan A/B test (80% treated / 20% control) untuk mengukur incremental uplift secara kausal.
